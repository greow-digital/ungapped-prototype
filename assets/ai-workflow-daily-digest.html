AI Workflow · Stand-out Element
Daily Performance Digest to Slack
Same data the Group Marketing dashboard pulls — but pushed to where work happens at 08:00 every morning. Five MCP-connected sources, n8n orchestration, Claude as the synthesis layer. Costs ~5 SEK/day in API calls and saves the 12-minute morning data-pull ritual across the team.
Schedule trigger fires 08:00 daily → AI Workflow runs against 5 paid + analytics sources via MCP, with Claude as the LLM → digest posted to #group-marketing + DM to CMO with interactive buttons.
Daily flow · 08:00 every morning
1
Cron n8n trigger fires at 08:00 CET
Single workflow run · no manual intervention. Workflow runs in <90 seconds end-to-end. If any source fails the digest still ships with a fallback note.
2
MCP Parallel fetch from 5 sources
Each MCP returns yesterday's data scoped to the group's accounts. Brand mapping happens via a lookup table (campaign-name regex → brand). Currency normalized to SEK.
3
Claude Synthesis with 7-day context
Claude receives normalized data + last 7 daily digests as context. Computes deltas vs prior day and 7-day avg. Detects anomalies (CTR drop, CPC spike, conversion gap) and writes flags in plain language.
4
Claude Format as Slack Block Kit
Claude outputs structured JSON matching Slack's Block Kit schema · sections, bullet lists, dividers, action buttons. No regex post-processing — schema is enforced via a tool definition.
5
Slack Post to #group-marketing + DM to CMO
Channel post for transparency, DM for the CMO so it shows up in personal feed. Buttons trigger sub-workflows: "Open dashboard", "Diagnose flag", "Snooze tomorrow's digest".

01 · Why daily not weekly
Habits compound — weekly slips, daily sticks
Daily creates the muscle of looking at performance every morning. Weekly digests get ignored after sprint 2. The cost difference is <30 SEK/month, the behavior difference is everything.
02 · Why push not pull
Meet people where they already work
Even a great dashboard requires opening a tab. Slack is the first thing people check. Pushing data into Slack — formatted, synthesized, with flags — converts a 12-minute ritual into a 90-second scan.
03 · Why Claude as the layer
Synthesis > aggregation
A SQL pipeline can aggregate. Only an LLM can write "LinkedIn CTR fatigued — top ad set running 14 days, replace it" instead of "CTR 0.4%". The flags are the value, not the numbers.
